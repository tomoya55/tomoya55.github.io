Hybrid Storage of Graph DB and RDB on Rails (Version 1)

03 Jun 2015

This is my first trial to integrate Graph DB and RDB on one Rails project, and codes are not smart enough.

In my project, I had become to handle complex relations of things. Since I was aware of the existence of Graph DB, it was a good chance to try Graph DB in my project.

This article describes how I have tackled a hybrid system of Graph DB and Relational DB on a Rails project.

I will use Neo4j as a Graph DB and MySQL as a RDB. Both middlewares are very popular and have great support for Rails.

Why Hybrid?

The bottomline is that I am very new to Graph DB, so I don’t want to move everything into Graph DB. I prefer to introduce Graph DB on only where it’s specialised in: Relationships of nodes.

Step 1 Defining models

As a newbie to Neo4j, I found neo4jrb/neo4j gem is very promising because it’s an active-model wrapper, so I can use APIs I’ve familiar with.

Let’s start with this gem!

# Gemfile
source "https://rubygems.org"
gem "rails", "4.2.1"
gem "mysql2"
gem "neo4j", "~> 5.0.0.rc2"

then, after bundle install, let’s create two models: Show and Band.

bundle exec rails g model Band
bundle exec rails g model Show

also we need models for neo4j

bundle exec rails g neo4j:model Graph::Band
bundle exec rails g neo4j:model Graph::Show

Step 2 Creating records

What I like to do is when I create a new Band on MySQL, it will automatically create a new node on Neo4j as well. let’s use after_commit callback to do it automatically. Also we like to share the same id between MySQL and Neo4j, so we need to override the default uuid property with the MySQL id.

module Graph
  module Resource
    extend ActiveSupport::Concern

    included do
      after_commit on: [:create, :update] do
        node.attributes = as_graph_json
        node.save!
      end

      after_commit on: [:destroy] do
        node.destroy
      end
    end
  end

  module ActiveRecordResource
    extend ActiveSupport::Concern

    included do
      attr_accessor :resource_id
      id_property :uuid, on: :resource_id
    end
  end

  module ClassMethods
      def graph_class
        "Graph::#{name}".constantize
      end
    end

    def node
      @_node ||= begin
        persisted? && graph_class.find_by_id(id) || graph_class.new
      end
    end

    def graph_class
      self.class.graph_class
    end

    def as_graph_json
      {resource_id: id}
    end
  end
end

class Band < ActiveRecord::Base
  include Graph::Resource
end

class Show < ActiveRecord::Base
  include Graph::Resource
end

module Graph
  class Band
    include Neo4j::ActiveNode
    include ActiveRecordResource
  end
end

module Graph
  class Show
    include Neo4j::ActiveNode
		include ActiveRecordResource
  end
end

Step 3 Creating relations

Well, then let’s define relationships between models. We will let Neo4j handle relations, so we need to define them only in Neo4j models.

module Graph
  class Band
    include Neo4j::ActiveNode
    include ActiveRecordResource

    has_many :out, :shows, type: "ActsIn"
  end
end

module Graph
  class Show
    include Neo4j::ActiveNode
    include ActiveRecordResource

    has_many :in, :bands, origin: :shows
  end
end

and try it from rails console

> band = Band.first
# SELECT  `bands`.* FROM `bands`  ORDER BY `bands`.`id` ASC LIMIT 1
> show = Show.first
# SELECT  `shows`.* FROM `shows`  ORDER BY `shows`.`id` ASC LIMIT 1

My 2 cents

My first trial on hybrid system of neo4j and active record seems working, however, there are lot of improvements before bringing it to the real word.

  1. Don’t like to have 2 model files of Band and Graph::Band. should be unified
  2. Switching AR world and Neo4j world is very tedious. We don’t like to switch the context like band.node

The reason I had to separate model files are that neo4j gem is an wrapper of ActiveModel, so lots of apis will be conflicted if we share the same model.

class Band < ActiveRecord::Base
  include Neo4j::ActiveNode
end

Band.create(name: "rolling stones")

Above the snippet, the original create defined in ActiveRecord::Base is overridden by Neo4j::ActiveNode, so I won’t work.

The next trial will be to integrate both models into one by removing neo4j gem and accessing lower level of api to interact with neo4j.
